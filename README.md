# LUMIA Intern

User-facing template for running personal Codex interns connected to the shared Feishu relay.

Each user should fork this repo, clone their own fork, and run interns from that clone. Intern metadata is committed to `workspace/interns/<intern_name>/`, so changes generated by interns stay in the user's own repo and can be pushed normally.

## What Is Included

- `lumia-intern.sh`: single entry point for setup/start/list/status/stop/delete.
- `intern-cli/`: local daemon, user lifecycle commands, and Codex session startup.
- `vscode-extension/hooks/`: Codex hook runtime used by the CLI; no VS Code plugin install is required.
- `workspace/`: clean metadata root. New interns are created under `workspace/interns/`.

Admin-only relay server code, VSIX releases, historical tasks, tests, visualizer code, deprecated code, and runtime secrets are intentionally not included.

## Prerequisites

Install or prepare these before first run:

- `git` with push access to your fork.
- `tmux`.
- Python 3.10 or newer.
- Codex CLI, already authenticated by the user on this machine.
- Feishu relay URL/token from the administrator.
- Your Feishu mobile phone number.

Install Python runtime packages in your chosen environment:

```bash
python -m pip install -r intern-cli/requirements.txt
```

Check that your fork can be pushed:

```bash
git remote -v
git push --dry-run origin HEAD:$(git branch --show-current)
```

## First Run

From your fork/clone:

```bash
cd ~/LUMIA-intern
./lumia-intern.sh start --intern intern_research_bot --relay-url ws://<relay-host>:28081
```

The script will:

1. Use `WORK_AGENTS_ROOT=$HOME` by default.
2. Use this clone as the project repo.
3. Create a project id named `<username>_project`.
4. Check git read/push access.
5. Configure relay connection under `$HOME/.feishu_registry/_owner.json`.
6. Start or reuse the local daemon.
7. Create and push `workspace/interns/<intern_name>/` metadata.
8. Start Codex in tmux session `<intern_name>`.

If the administrator has provided a fixed public relay URL, you can avoid typing it each time:

```bash
export INTERN_DEFAULT_RELAY_URL=ws://<relay-host>:28081
./lumia-intern.sh start --intern intern_research_bot
```

The relay token is secret. Pass it with `--relay-token`, set `INTERN_RELAY_TOKEN`, or enter it when prompted. Do not commit it.

## Daily Commands

```bash
# Add or restart an intern
./lumia-intern.sh start --intern intern_coding_bot

# List configured interns and tmux sessions
./lumia-intern.sh list

# Show local daemon and relay config status
./lumia-intern.sh status

# Stop an intern Codex/tmux session
./lumia-intern.sh stop --intern intern_coding_bot

# Delete a stopped intern
./lumia-intern.sh delete --intern intern_coding_bot
```

Delete prints the exact local/project/Feishu resources it will remove and asks for confirmation. It does not stop or delete the daemon, relay config, Codex auth, or the project repo.

## Local Files Users Must Not Commit

These are ignored by `.gitignore` and should stay local:

- `key.txt`
- `.feishu_registry/`
- `.codex/`
- `.agents/`
- `llm_intern_logs/`
- `root/`
- `.intern_sessions.json`
- `enterprise/`

Codex authentication is intentionally user-local. Users should log in themselves; never package another user's `.codex` directory into the template.
